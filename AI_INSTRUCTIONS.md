# 智能菜谱系统 (Intelligent Recipe System) - 用户手机端 (H5) 开发指南

## 1. 项目概述
本项目是“智能菜谱系统”的面向普通用户的手机端应用（移动端H5网页）。
用户手机端主要是面向普通用户可以通过微信公众号进入使用 。首先，用户可以通过登录注册创建账号，并在个人中心里填好自己的健康状况和营养目标。吃饭前，既可以用关键字去菜谱检索，也可以让系统根据你的健康需求进行智能的菜品推荐 。吃完后，用户可以在饮食记录里记一笔，觉得好的菜谱就点个菜谱收藏，下次还能快速找回。最后，大家还可以在美食圈子里发动态交流经验，并通过菜谱评论模块对系统给出的菜谱好不好用给个真实的反馈评分

当前开发阶段声明：目前处于纯前端页面开发和交互还原阶段，暂不对接真实后端，数据采用前端 Mock 或写死的 JSON 数据。

## 2. 技术栈约束与规范
在生成、修改代码时，请严格遵守以下技术栈：
- **核心框架:** Vue 3 (强制使用 Composition API 和 `<script setup>` 语法)
- **构建工具:** Vite （使用pnpm进行依赖管理）
- **UI 组件库:** NutUI 4.x (京东移动端组件库，优先使用其内置组件还原设计)
- **CSS 方案:** Tailwind CSS (仅用于辅助 NutUI 无法覆盖的特殊布局、间距和基础排版)
- **路由管理:** 通过Vue Router构建单页面应用
- **网络请求:** Axios (统一放置在 `src/api` 目录下)
- **图标使用:** 优先使用 NutUI 内置的 Icon 图标，也可以利用unplugin-icons插件引入其他图标库（如mdi等）。

## 4. 后端接口信息
- **后端服务地址:** http://192.168.50.100:8082
- **请求方式:** 后端所有接口均为 POST 请求，接收的参数为 JSON 格式
- **请求头:** 在请求头中添加 Authorization 字段，值为 Bearer + token，其中token 用户登录或注册时返回的accessToken，若未登录则为空，如：Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJhZG1p

```json5
//后端接口相应结构
{
    //状态，0：成功，1000：操作失败，10001：用户未登录
    "code":0,
    //消息，成功返回 'success', 失败则返回具体错误消息，如："用户名或密码错误"
    "message":"success",
    //数据，返回的数据结构，当前示例返回的是对象，但它可能是一个数字，也可能是一个对象，也可能是一个数组，也可能是一个字符串，或者其他任意类型数据，不同接口返回的数据结构不同（在Java代码中通过泛型实现）
```
